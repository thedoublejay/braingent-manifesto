# Content Style

Write records for future retrieval, not narrative polish.

## Style

- Be factual and specific.
- Prefer short sections with clear labels.
- Put important identifiers near the top.
- Use exact names, versions, commits, PRs, tickets, and dates when known.
- Mark unknowns as `Unknown`.
- Link related records.
- Separate fact from inference.

## Good Durable Notes Include

- the original goal
- the actual outcome
- why key decisions were made
- what evidence was used
- what failed or surprised you
- what future agents should do differently

## Avoid

- long chat transcripts as final memory
- vague claims without root cause or evidence
- filenames whose meaning depends entirely on their folder
- copying sensitive data into memory
- local machine paths in public docs
